wordpress上传文件报错的解决方法（413 Request Entity Too Large、超过upload_max_filesize文件中定义的php.ini值）

发表于 2021-01-21 更新于 2021-08-17 分类于 程序设计 ， 网站

413 Request Entity Too Large问题nginx是限制上传大小，解决方法如下:1. 打开nginx配置文件 nginx.conf, 路径一般是：/etc/nginx/nginx.conf。2. 在http{}段中加入 client_max_body_size 64m; 64m为允许最大上传的大小。3. 保存后重启nginx，service nginx restart
